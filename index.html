1 — Short Summary / Elevator Pitch

Lambda (Vertical Entity for Neural Terminal Awareness) simulates a CLI horror/spy/puzzle game with sensor-driven tasks, face recognition, local progress storage, and modular commands.

2 — Purpose & UX Goals

  • Authentic CLI horror-terminal experience.
  • Sensor-based interactions (camera, mic, torch, vibration, battery).
  • Modular "tasks → tests → unlocks" flow, potential ARG-style storylines.

Player Flow: Launch → Accept warnings → Start PHYNEMO → Stage 1 Physical Tests → Stage 2 Emotional/Intellectual Tests → Optional Face Recognition → Unlock Mindmirror.

3 — File Structure & Key Code

  • Terminal UI: printAnimated(), createInputLine(), handleCommand()
  • Horror Story: startLambdaHorrorStory(), switchToHorrorMode(), showWarning()
  • Task System: taskManager(), completeTask(), resetAllTasks()
  • Physical Tests: cameraDarknessTask(), micLineGraphTask(), startLambdaTriggerMode()
  • Face Recognition: faceRecognitionTask(), selfieVerificationTask()
  • Utilities: captureFullTerminal(), downloadRealTimeCountdownLauncher()
  • LLM Integration: handleCMD() (Gemini API, remove key)

4 — Key Commands

  • launch -Venta — start horror story
  • ./activate_test_runner --init phy&emo — PHYNEMO protocol
  • task-01 … task-04 — physical tasks
  • start face / panel face / snapshot / record video — face recognition
  • voice-task / sound-detection / cream — mic-based challenges
  • possess / diagnose — glitch/narrative effects
  • help — full command list

5 — Face Recognition & ID Workflow

Camera → capture → descriptor → name/password → save to localStorage → callbacks. Face data stored locally; consent required for remote storage.

6 — Device & Sensor Interactions

  • Camera — face recognition, glitch feed, darkness tasks
  • Microphone — voice/scream tasks, sound detection
  • Flashlight / Torch — limited support, graceful fallback
  • Vibration — punishment/atmosphere effects, optional loops
  • Battery / Charger — gating final tests

7 — Security & Privacy Checklist

  • Remove hard-coded API keys (Gemini)
  • Face data: store locally, encrypt if stored remotely
  • Explain permissions clearly (camera, mic, notifications)
  • Flash content warning
  • Vibration loops optional / stoppable
  • Provide resetAllTasks and face data reset

8 — Running & Testing

  • Serve folder with live-server or python -m http.server
  • Modern mobile/desktop browser (mobile for flashlight/vibration)
  • Type help to see commands
  • Test face recognition: start face / panel face
  • Full flow: PHYNEMO → task-01 → task-04

9 — Next Steps / Improvements

  • Add consent dialogs & privacy policy
  • Modularize JS (terminal, audio, face recognition)
  • Accessibility & safety controls
  • Optional server-backed progress
